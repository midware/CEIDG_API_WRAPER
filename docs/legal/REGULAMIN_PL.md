# Regulamin leadbase.network

Wersja robocza: 2026-06-30  
Status: draft do przegladu prawnego przed publikacja.

## 1. Definicje

1. **Operator** - [UZUPELNIC: pelna nazwa firmy, adres, NIP/REGON/KRS lub CEIDG, email].
2. **Serwis** - platforma leadbase.network, API, panel uzytkownika, dokumentacja, mechanizmy tokenow oraz powiazane uslugi.
3. **Uzytkownik/Klient** - przedsiebiorca, osoba prawna, jednostka organizacyjna albo osoba dzialajaca zawodowo lub gospodarczo, korzystajaca z Serwisu.
4. **Konto** - indywidualny dostep do Serwisu zabezpieczony loginem, haslem i/lub kluczem API.
5. **API** - interfejs programistyczny pozwalajacy pobierac dane z bazy leadbase.network.
6. **Tokeny** - jednostki rozliczeniowe zuzywane przy zapytaniach API lub innych funkcjach platnych.
7. **Dane firmowe** - dane dotyczace przedsiebiorcow i firm, w szczegolnosci dane pochodzace z CEIDG lub innych wskazanych zrodel.
8. **Dane kontaktowe** - telefon, email, adres strony WWW lub inne dane pozwalajace skontaktowac sie z firma lub osoba prowadzaca dzialalnosc.

## 2. Charakter uslugi

1. Serwis jest narzedziem B2B do wyszukiwania, pobierania i analizowania danych firmowych przez API oraz panel uzytkownika.
2. Serwis nie jest urzedowym rejestrem i nie zastepuje CEIDG, KRS, GUS ani innych zrodel publicznych.
3. Operator doklada starannosci, aby dane byly aktualne i poprawne, ale nie gwarantuje ich kompletnosci, aktualnosci ani przydatnosci do konkretnego celu.
4. Serwis jest przeznaczony wylacznie dla uzytkownikow profesjonalnych/B2B. Korzystanie przez konsumentow jest wylaczone, chyba ze Operator wyrazi na to pisemna zgode.

## 3. Rejestracja i konto

1. Do korzystania z funkcji platnych lub API wymagane jest utworzenie konta.
2. Uzytkownik musi podac prawdziwe dane oraz potwierdzic adres email.
3. Uzytkownik odpowiada za bezpieczenstwo hasla, kluczy API oraz za wszystkie dzialania wykonane przy uzyciu jego konta.
4. Zabronione jest udostepnianie konta lub kluczy API osobom nieuprawnionym.
5. Operator moze odmowic rejestracji, zawiesic konto albo ograniczyc dostep, jezeli istnieje uzasadnione podejrzenie naruszenia prawa, regulaminu, bezpieczenstwa lub praw osob trzecich.

## 4. Tokeny, ceny i rozliczenia

1. Usluga dziala w modelu tokenowym. Koszt zapytania zalezy m.in. od liczby zwroconych rekordow i rodzaju pobieranych danych.
2. Aktualny cennik oraz pakiety tokenow sa publikowane w Serwisie lub w API.
3. Tokeny moga byc przyznane bezplatnie przy rejestracji, zakupione w pakietach lub przyznane promocyjnie.
4. Zuzyte tokeny nie podlegaja zwrotowi, chyba ze bezwzglednie obowiazujace przepisy prawa stanowia inaczej albo Operator indywidualnie przyzna reklamacje.
5. Operator moze zmienic cennik na przyszlosc. Zmiana nie odbiera tokenow juz zakupionych, chyba ze regulamin promocji stanowil inaczej.
6. Faktury sa wystawiane zgodnie z danymi podanymi przez Klienta.
7. W przypadku platnosci przez zewnetrznego operatora platnosci Klient moze podlegac regulaminom tego operatora.

## 5. Dozwolone korzystanie z danych

1. Klient moze korzystac z danych w legalnych celach gospodarczych, w szczegolnosci do analizy rynku, weryfikacji kontrahentow, przygotowania ofert B2B, segmentacji firm, wzbogacania wlasnego CRM i raportowania.
2. Klient jest samodzielnie odpowiedzialny za ustalenie podstawy prawnej dalszego wykorzystania danych, w tym kontaktu telefonicznego, emailowego, SMS lub marketingowego.
3. Sam fakt udostepnienia danych przez Serwis nie oznacza zgody osoby lub firmy na otrzymywanie informacji handlowych, marketingu bezposredniego ani kontaktu automatycznego.
4. Klient zobowiazuje sie przestrzegac RODO, przepisow o komunikacji elektronicznej, przepisow o zwalczaniu nieuczciwej konkurencji oraz innych przepisow majacych zastosowanie do jego dzialalnosci.

## 6. Zakazane zastosowania

Zabronione jest:

1. wysylanie spamu, masowych niezamowionych wiadomosci lub prowadzenie natarczywych kampanii;
2. kontaktowanie sie z osobami bez wymaganej podstawy prawnej lub zgody, jezeli jest wymagana;
3. wykorzystywanie danych do dyskryminacji, nękania, oszustw, podszywania sie lub dzialan bezprawnych;
4. odsprzedaz, sublicencjonowanie lub publiczne udostepnianie danych jako konkurencyjnej bazy bez zgody Operatora;
5. scraping, reverse engineering, obchodzenie limitow, rate-limitow lub zabezpieczen;
6. uzywanie wielu kont w celu omijania limitow lub oplat;
7. naruszanie praw osob, ktore zlozyly sprzeciw lub opt-out.

## 7. API i limity techniczne

1. Operator moze stosowac limity zapytan, rate-limiting, limity dzienne/miesieczne oraz mechanizmy antyfraudowe.
2. Klient nie moze generowac ruchu zaklocajacego dzialanie Serwisu.
3. Operator moze tymczasowo ograniczyc API w razie awarii, prac serwisowych, zmian po stronie zrodel danych, incydentu bezpieczenstwa lub naruszenia regulaminu.
4. Dokumentacja API ma charakter techniczny i moze sie zmieniac wraz z rozwojem produktu.

## 8. Dane, jakosc i odpowiedzialnosc

1. Dane moga pochodzic z rejestrow publicznych, zrodel oficjalnych lub innych jawnie wskazanych zrodel.
2. Dane moga byc nieaktualne, niekompletne, zdublowane lub bledne.
3. Klient powinien samodzielnie zweryfikowac dane przed podjeciem decyzji gospodarczej, prawnej, finansowej lub marketingowej.
4. Operator nie odpowiada za skutki wykorzystania danych przez Klienta niezgodnie z prawem, regulaminem lub dobrymi obyczajami.

## 9. Wlasnosc intelektualna

1. Serwis, API, dokumentacja, kod, znaki towarowe, layout i baza w zakresie chronionym prawem stanowia wlasnosc Operatora lub jego licencjodawcow.
2. Klient otrzymuje niewylaczne, niezbywalne prawo korzystania z Serwisu zgodnie z regulaminem.
3. Regulamin nie przenosi na Klienta praw do Serwisu, oprogramowania ani know-how Operatora.

## 10. Reklamacje i support

1. Reklamacje mozna skladac na adres [UZUPELNIC: support@leadbase.network].
2. Reklamacja powinna zawierac opis problemu, identyfikator konta, przyblizona date zdarzenia i oczekiwany sposob rozwiazania.
3. Operator odpowiada w rozsadnym terminie, co do zasady do 14 dni roboczych.

## 11. Zmiany regulaminu

1. Operator moze zmienic regulamin z waznych przyczyn, w szczegolnosci zmian prawa, zmian funkcji Serwisu, cennika, bezpieczenstwa lub modelu biznesowego.
2. O istotnych zmianach Operator poinformuje uzytkownikow z wyprzedzeniem, np. emailem lub komunikatem w panelu.
3. Dalsze korzystanie z Serwisu po wejscu zmian w zycie oznacza akceptacje nowego regulaminu.

## 12. Prawo wlasciwe i spor

1. Regulamin podlega prawu polskiemu.
2. Spory beda rozstrzygane przez sad wlasciwy dla siedziby Operatora, o ile bezwzglednie obowiazujace przepisy nie stanowia inaczej.
